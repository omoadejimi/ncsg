# How to contribute to the Nigerian Content Style Guide

You don't need to know how to code to improve this guide. If you can write
clearly and you care about good content for Nigerian users, you can
contribute. This page explains how, in plain steps.

Everything happens on GitHub, the website where the guide's content lives.
You'll need a free GitHub account — sign up at [github.com](https://github.com)
if you don't have one. It takes two minutes.

---

## The big picture (read this first)

The guide is open. Anyone can suggest a change. But nothing you submit goes
live straight away — an editor reviews it first. So you can never break the
site, and you don't need to ask permission before suggesting something. Make
your change, send it in, and an editor takes it from there.

A suggested change is called a **pull request**. Don't let the name worry you;
it just means "here's a change I'd like you to consider."

---

## Fixing a typo or rewording a sentence

This is the most common contribution and the easiest. You can do it entirely
in your browser.

1. Go to the page on the live site that has the problem.
2. Scroll to the bottom and click **"Suggest an edit on GitHub."**
3. GitHub opens the page's text. If it asks you to "fork" the project, click
   the green button to agree — this just makes your own copy to edit safely.
4. Click the pencil icon to edit. Find the text and change it.
5. Scroll down. In the box, write one line saying what you changed, for
   example: "Fixed typo in error messages page."
6. Click **Propose changes**, then **Create pull request** on the next screen.

Done. An editor will see it, and you'll get an email when it's reviewed.

## Understanding the formatting

The content is written in a simple format. You only need a handful of things:

| To do this | Write this |
| --- | --- |
| A section heading | `## My heading` |
| A smaller heading | `### My smaller heading` |
| **Bold text** | `**bold text**` |
| A bullet point | Start the line with `- ` |
| A link | `[words people see](the-web-address)` |

You'll also see special blocks already in the pages. Copy their shape when you
want to add your own.

A "Do and Don't" pair looks like this:

```
<DoDont>
<Do>
Send ₦5,000
</Do>
<Dont>
Submit
</Dont>
</DoDont>
```

A note box looks like this:

```
<Note>
Something helpful the reader can skip if they're in a hurry.
</Note>
```

A warning box looks like this:

```
<Warning>
Something important that prevents a mistake.
</Warning>
```

The safest way to add any of these is to find one already in a page, copy it,
and change the words inside.

## Adding a new example

Real examples are the most valuable thing you can give the guide. If you've
seen copy in a Nigerian product that worked well or badly, add it as a
Do/Don't pair on the relevant page.

Two rules:

- Change any details that could identify a real customer or company. Use made-
  up names and round figures.
- Make each Do/Don't pair show **one** idea, not several.

---

## Suggesting bigger changes

If you want to add a whole new page, or change a recommendation the guide
currently makes, don't just edit — start a conversation first. Go to the
repository, click the **Issues** tab, and click **New issue**. Describe:

- the problem you've noticed,
- what you think the guide should say instead,
- and any evidence (user research, support tickets, testing) behind it.

An editor will discuss it with you before any change is made. This keeps the
guide consistent and evidence-based.

---

## The golden rules

- **You can't break anything.** Every change is reviewed before it goes live.
- **You don't need permission to suggest.** Just send it in.
- **Ask if you're stuck.** Open an issue, or mention in your pull request that
  it's your first time — someone will help.
- **Be kind.** We discuss ideas and evidence, never attack people. See the
  Code of conduct page for the full version.

---

## What happens after you submit

1. An editor reads your change, usually within two weeks.
2. They might accept it as-is, suggest small adjustments, or explain why it
   doesn't fit — always with reasons.
3. Once accepted, your change goes live automatically, and you're credited.

Thank you for helping make digital products clearer for Nigerian users.
